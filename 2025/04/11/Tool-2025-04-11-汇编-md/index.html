汇编学习

声明：我所研究的汇编是win32位的MASM汇编，并不是传统的8086汇编！

一、汇编的基本语法知识

  1. 模式定义

    这些指令定义了程序使用的指令集、工作模式和格式，接下来我们来逐个介绍每一个语句是什么含义：

    • .386
      （伪指令）告诉编译器在本程序中使用的指令集。
      （若是要写驱动程序，定义.386p指在特权指令0级上运行）

    • .model flat, stdcall
      .model在低版本宏汇编中已经存在，用来定义程序工作模式
      基本格式：.model 内存模式, [语言模式][其他模式]
      内存模式定义能够影响最后生成的可执行文件：

      内存模式 各种模式的介绍
      tiny 建立 .com文件，所有 .code, .data, .stack全部在64KB的段内
      small 建立 .code, .data分别用一个64KB的段的 .exe文件
      medium .code能有很多个64KB段，.data只有一个
      compact .data能有很多个64KB段，.code只有一个
      large .code和 .data都能有很多个64KB段
      huge 和large相同，.data中一个数组可以超过64KB
      flat win32使用的模式，.code，.data共同使用一个4GB的段

      win32只有flat模式！

      .model flat，还需要指定语言模式，例子使用的是 stdcall，它指出了调用子程序或 win32 API时参数传递的次序和堆栈平衡的方法。

      除了 stdcall 以外还有 C, SysCall, BASIC, FORTRAIN, PASCALL
      高级语言都是通过堆栈来传递参数， win32 API调用使用的是 stdcall参数

      1. _stdcall调用
        _stdcall时Pascal程序的缺省调用方式，参数采取从右至左的压栈方式，被调用的函数自身在返回前清空堆栈。 win32 API采取 _stdcall调用方式
        example:

        f(x,y,z)f(x, y, z)f(x,y,z) zzz第一个入栈，yyy其次，最后是xxx。 函数返回时自己清空堆栈。
      2. _cdecl调用
        是 C/C++ 的缺省调用方式，参数从右往左压入栈，内存栈由调用者维护，函数返回时不会自己清空堆栈，只能被 C/C++ 调用

    • option casemap:none
      option:none 是告诉编译器，程序变量名和子程序变量名对大小敏感。

二、基本指令

  1. 数据传送指令
    • mov
      mov eax, 33221100H 立即数寻址
      mov ebx, eax 寄存器寻址
      mov ecx, ds:[405000H] 存储器寻址 基本格式：
      mov 目的操作数， 源操作数
    • movzx
      全零扩展并传送
      movzx reg32, reg/mem8
      movzx reg32, reg/mem16
      movzx reg16, reg/mem8
    • movsz
      符号扩展并传送
      movsx reg32, reg/mem8
      movsx reg32, reg/mem16
      movsx reg16, reg/mem8
    • xchg
      xchg reg, reg/mem
      xchg reg / mem, reg
    • lea
  2. 算术运算指令
    1. 加法和减法
    • inc/dec
      inc /dec dst操作数+/-1
    • add/sub
      add / sub dst, src
    • neg
      求补 neg dst
    • cmp
      cmp dst, src
    • adc/sbb
      带进位加法和带错位减法

加减法影响的标志位：

  • cf置1：无符号整数溢出
  • of置1：有符号整数溢出
  • zf置1：意味着操作结果为0
  • sf置1：意味着操作产生的结果为负数
  1. 乘法与除法
  • mul
    mul reg/mem8
    mul reg/mem16
    mul reg/mem32
  • imul
    执行有符号整数乘法
  • div
    无符号除法指令
  • idiv
    有符号除法指令
  1. 转移指令
    1. 地址转移指令
      • and, er, xor, not, test, cmp
    2. 跳转指令
      • jmp指令
        • jc 进位跳转 cf = 1
        • jnc 无进位跳转 cf = 0
        • jz 为零跳转 zf = 1
        • jnz 非零跳转 zf = 0
        • js 有符号跳转 sf = 1
        • jns 无符号跳转 sf = 0
        • jp 偶校验跳转 pf = 1
        • jnp 奇校验跳转 pf = 0
        • je 相等则跳转
        • jl 小于则跳转
        • jg 大于则跳转
      • 布尔指令
        • and, or, xor, not
        • test：一种非破坏性的and操作

三、过程（函数）

  1. 汇编语言堆栈
    • 栈寄存器：esp(栈指针), ebp
    • 栈操作：push, pop
  2. 汇编语言过程
    • 过程定义：proc / endp
    • 过程调用与返回：call, ret
    保存的寄存器 <- ESP
    局部变量n
    …
    局部变量1 <- -4
    被保存的ebp <- EBP
    返回地址 <- +8
    参数1 <- +0ch
    …
    参数n-1
    参数n
    • 参数 [ebp+n]
    • 局部变量 [ebp-n]

四、数组

  1. 基址-变址：将两个寄存器（称为基址和变址）相加，生成一个偏移地址
    [base + index]
  2. 基址-变址-偏移量
    [base + index + displacement]
    displacement[base + index]

五、Win32汇编

  1. MS-Windows 汇编编程
    • 动态链接库（DLL）：系统功能模块
    • API：应用程序编程接口，系统以API的形式提供功能调用
    • 调用约定：WINAPI， 即__stdcall
    • Win32 API函数不保存 EAX, EBX, ECX 和 EDX
  2. Win32 常用库
    • kernel32: 操作系统内核相关，如文件，内存，进程
    • user32: 用户界面相关，如窗口，消息等

x64汇编编程

  • 64 位 MASM 不支持 .model 伪指令
  • 64 位 MASM 中，使用 PROTO 关键字的语句不带参数
  • 64 位 MASM 不支持 INVOKE 伪指令
