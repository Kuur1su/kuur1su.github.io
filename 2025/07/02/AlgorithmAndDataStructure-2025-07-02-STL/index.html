C++ STL 容器手册

顺序容器 (Sequence Containers)

std::vector

  • 简介： std::vector 是一种动态数组容器，元素在内存中连续存放，可通过下标随机访问。它自动扩展容量，以存储更多元素。由于连续存储，随机访问（[] 或 at）时间复杂度为 O(1)；对末尾元素进行添加或删除（push_back/pop_back）摊还为 O(1)，在中间插入或删除元素平均为 O(n)。

  • 常用操作：

    • 构造：vector<T> v;、vector<T> v(n, val);、vector<T> v = {a,b,c};
    • 增删：v.push_back(x); // 尾部添加，摊还O(1)；v.pop_back(); // 删除末尾，O(1)；v.insert(it, x); // 在迭代器 it 前插入，O(n)；v.erase(it); // 删除 it 指向元素，O(n)；v.clear(); // 清空，O(n)。
    • 访问：v[i] 或 v.at(i) 随机访问元素，O(1)；v.front(), v.back() 访问首尾元素，O(1)；v.begin(), v.end(), v.size(), v.empty() 等为常数时间操作。

  • 竞赛场景： 常用来存储需要动态增长的数组、线性序列。典型用法包括读入数据后排序、坐标压缩、动态规划状态存储等。由于支持快速随机访问和顺序遍历，适合按索引或迭代操作。可使用 reserve() 预先分配内存以提高效率。

  • 示例： 使用 vector 读入整数并排序、访问元素。

std::deque

  • 简介： std::deque 是一种双端队列容器，支持在两端快速插入删除，同时也允许随机访问。内部通常用多个连续的存储块构成，插入末端无需像 vector 那样移动所有元素。复杂度：随机访问 O(1)，两端插入/删除 O(1)，在中间插入/删除 O(n)。

  • 常用操作：

    • 构造：deque<T> dq;、deque<T> dq(n, val);
    • 两端增删：dq.push_back(x); // 尾部添加，O(1)；dq.pop_back(); // 删除末尾，O(1)；dq.push_front(x); // 头部添加，O(1)；dq.pop_front(); // 删除头部，O(1)。
    • 中间插入：dq.insert(it, x); // O(n)；dq.erase(it); // O(n)；其他如 begin(), end(), size(), clear() 复杂度为 O(1) 或 O(n)（clear() 通常线性）。

  • 竞赛场景： 常用于需要在两端同时进行操作的情况，如滑动窗口（维护固定窗口内的最值或其他信息）、广度优先搜索（BFS）的队列实现、以及需要在头尾插入的算法（比如双指针）。由于可以前后端操作且支持快速随机访问，比 list 更常用。

std::list

  • 简介： std::list 是双向链表容器，支持常数时间在任意位置插入或删除元素，但不支持随机访问，访问元素需要线性时间（O(n)）。典型实现为双链表，每个元素存储前后指针。

  • 常用操作：

    • 构造：list<T> lst;
    • 插入删除：lst.push_back(x);、lst.push_front(x);、lst.insert(it, x); // 在迭代器 it 前插入，O(1)、lst.erase(it); // 删除 it，O(1)。
    • 访问：只能用迭代器顺序遍历（begin(), end()），size() 通常是常量时间（C++11 起要求）。clear() 线性时间。front(), back() 访问首尾，O(1)。注意 [] 不可用。

  • 竞赛场景： 一般不如 vector 或 deque 常用，因为链表遍历开销大。但当需要在序列中间频繁插入/删除而不关心随机访问时可用。例如维护一个活动序列，每次在当前位置插入或删除元素时可使用。list 还支持 splice 等操作，可将整个链表片段快速移动到另一个链表中（耗时O(1)）；在某些题目中用于高效地拼接链表。

容器适配器 (Container Adaptors)

std::stack

  • 简介： std::stack 是栈适配器，按照后进先出（LIFO）原则组织元素。默认底层容器为 deque（也可指定为 vector 或 list）。对栈顶元素进行插入、删除、访问操作，其他操作被禁止。

  • 常用操作：

    • stack<T> st;
    • st.push(x); // 向栈顶插入元素，O(1)
    • st.pop(); // 删除栈顶元素，O(1)
    • st.top(); // 访问栈顶元素，O(1)
    • st.empty();、st.size();
    • 注意：push、pop 都是常数时间（底层容器 deque 或 vector 提供）

  • 竞赛场景： 用于后进先出逻辑，如深度优先搜索（DFS）的显式栈实现、括号匹配、处理运算符优先级时的运算符栈、递归转换为非递归等。

  • 示例： 栈的基本用法。

std::queue

  • 简介： std::queue 是队列适配器，按照先进先出（FIFO）原则组织元素。默认底层容器为 deque（也可指定为 list）。常用作广度优先搜索（BFS）或简单的队列结构。

  • 常用操作：

    • queue<T> q;
    • q.push(x); // 在队尾插入，O(1)
    • q.pop(); // 删除队头元素，O(1)
    • q.front(); // 访问队头元素，O(1)
    • q.back(); // 访问队尾元素，O(1)
    • q.empty();、q.size();

  • 竞赛场景： 常用于图的 BFS、拓扑排序、滑动窗口等。所有操作均为常数时间。

  • 示例： BFS 遍历二叉树或图的层次顺序。

std::priority_queue

  • 简介： std::priority_queue 是优先队列适配器，本质上是大顶堆（默认）或小顶堆。默认使用 std::less，获取最大元素；可自定义比较器得到最小元素堆。查找最大元素 (top()) 为常数时间 O(1)，插入 (push) 和删除堆顶 (pop) 为对数时间 O(log n)。底层一般使用 vector + make_heap。

  • 常用操作：

    • priority_queue<T> pq; 或 priority_queue<T, vector<T>, greater<T>> minPq;
    • pq.push(x); // 插入元素，O(log n)
    • pq.pop(); // 删除堆顶元素，O(log n)
    • pq.top(); // 访问当前最大（或最小）元素，O(1)
    • pq.empty(); q.size();

  • 竞赛场景： 常用于处理需要频繁取最大/最小的场景：贪心算法（如每次取最大的/最小的项目）、最短路算法（Dijkstra，取当前最短距离顶点）、合并多个有序序列（多路归并）、事件模拟等。

  • 示例： 插入若干数并输出从大到小的顺序。

关联容器 (Associative Containers)

std::set

  • 简介： std::set 是有序关联容器，存储唯一键值，自动按键升序排序。通常基于红黑树实现，因此插入、删除、查找等操作平均时间复杂度为 O(log n)。

  • 常用操作：

    • 构造：set<T> s;、set<T> s = {a,b,c};
    • 插入：s.insert(x); // 插入元素，若已存在则忽略，O(log n)
    • 删除：s.erase(x); // 删除键为 x 的元素，O(log n)
    • 查找：auto it = s.find(x); // 查找 x，返回迭代器，O(log n)
    • 访问：*s.begin() 得最小元素，*prev(s.end()) 得最大元素。支持双向迭代器遍历。lower_bound(k), upper_bound(k) 在有序集合中进行二分查找。
    • 其他：s.size(), s.empty(), s.clear()。

  • 竞赛场景： 常用于需要动态维护一组有序数据的场景：去重并自动排序、求第 k 大元素、动态统计范围等。比如维护窗口中的元素，用于“滑动窗口最值”；或者作为动态的排序集，能高效查找下界、上界。 multiset 类似但允许重复元素。

  • 示例： 插入元素并遍历输出排序结果。

std::multiset

  • 简介： std::multiset 与 set 类似，但允许重复键。内部也是有序存储，基于红黑树。插入、删除、查找操作时间复杂度同样为 O(log n)。

  • 常用操作：

    • 构造：multiset<T> ms;
    • 插入：ms.insert(x); // 可插入重复元素，O(log n)
    • 删除：ms.erase(x); // 删除键等于 x 的所有元素或通过迭代器删除单个元素，O(log n)
    • 查找：ms.find(x); // 查找等于 x 的一个元素，O(log n)；ms.count(x); // 统计等于 x 的个数，通常也O(log n)；ms.lower_bound(x)/upper_bound(x) 获取范围。

  • 竞赛场景： 可用于需要统计频次但保持排序的场景，如维护滑动窗口中多元素的有序集合、模拟多重优先队列等。比如求序列的中位数时可用两个 multiset 动态平衡。

std::map

  • 简介： std::map<Key,T> 是有序关联容器，存储键值对 (key,value)，键唯一并按键排序。底层通常用红黑树实现，插入、删除、查找（按键）等操作 O(log n)。提供按键快速查找、自动排序功能。

  • 常用操作：

    • 构造：map<Key,Val> mp;
    • 插入/访问：mp[key] = val; // 插入或修改键 key 对应的值，O(log n)；mp.insert({key, val}); // 不覆盖已存在的键。
    • 查找：auto it = mp.find(key); // 找到返回迭代器，否则 end(), O(log n)
    • 删除：mp.erase(key); // 删除键为 key 的元素，O(log n)
    • 迭代：按键升序遍历 (i->first, i->second)。lower_bound(key)、upper_bound(key) 等同 set。

  • 竞赛场景： 常用于需要按键快速查找的映射，如统计频率（key 为元素，value 为频率）、离散化（坐标压缩）、维护键值映射等。因其键有序，还可用于需要按顺序访问键值对的场景，如找前驱/后继键。

  • 示例： 使用 map 统计整数出现频率。

std::multimap

  • 简介： std::multimap<Key,T> 与 map 相似，但允许键重复。内部有序存储，插入、查找等操作为 O(log n)。常用于一个键关联多个值的情况。

  • 常用操作：

    • 构造：multimap<Key,Val> mm;
    • 插入：mm.insert({key, val}); // 插入一个键值对，O(log n)，可以插入相同 key。
    • 查找：auto range = mm.equal_range(key); // 返回等键元素的区间 [lb, ub)；mm.find(key); // 返回指向其中一个元素的迭代器。
    • 删除：mm.erase(key); // 删除所有键为 key 的元素 或使用迭代器删除单个元素。

  • 竞赛场景： 较少单独使用，通常通过 map<key, vector<Val>> 实现类似功能。不过在需要键值自动排序且允许重复键时可用。

无序关联容器 (Unordered Associative Containers)

std::unordered_set

  • 简介： std::unordered_set 是基于哈希表的容器，存储唯一键值，不保证元素顺序。查找、插入、删除在平均情况下为 O(1)，最坏情况下为 O(n)。由于哈希特性，元素散列后快速定位，但没有排序功能。

  • 常用操作：

    • 构造：unordered_set<T> us;
    • 插入：us.insert(x); // 插入元素，平均O(1)
    • 删除：us.erase(x); // 删除值为 x 的元素，平均O(1)
    • 查找：us.find(x); // 返回迭代器，平均O(1)；us.count(x); // 返回 0/1
    • 不支持按顺序访问（迭代器遍历为散列顺序）。其他有：bucket_count(), load_factor() 等。

  • 竞赛场景： 常用于快速判断元素是否存在、去重、处理哈希集合问题。与 set 相比，当不需要有序时效率更高。适合需要频繁查找（membership test）的场合，如判断重复、记录访问过的状态等。

std::unordered_map

  • 简介： std::unordered_map<Key,T> 是基于哈希表的键值对容器，键值对无序存储。平均情况下插入、查找、删除复杂度为 O(1)，最坏情况下为 O(n)。与 map 区别在于速度更快但不排序。

  • 常用操作：

    • 构造：unordered_map<Key,Val> um;
    • 插入/访问：um[key] = val; // 插入或修改，平均O(1)；um.insert({key,val}); // 若键已存在则跳过。
    • 查找：auto it = um.find(key); // 平均O(1)
    • 删除：um.erase(key); // 平均O(1)
    • um.size(), um.clear()。由于无序，不支持按键遍历顺序。

  • 竞赛场景： 常用于需要快速映射关系、统计频率、记忆化缓存等场景。当键为任意类型（需提供哈希函数）且顺序不重要时使用。如统计元素出现次数、哈希表判重、缓存中间结果、实现自定义哈希等。

  • 示例： 使用 unordered_map 统计字符串频率。

常见容器比较 (Container Comparisons)

  • set vs unordered_set：

    • set 使用红黑树实现，元素有序（可用 lower_bound 等），插入/查找/删除时间 O(log n)；
    • unordered_set 使用哈希表，无序，平均时间 O(1)（最坏情况 O(n)）；
    • 如果需要维护有序序列或前驱/后继功能，用 set，否则单纯判断存在与否常用 unordered_set（速度更快）。

  • map vs unordered_map：

    • map 是有序关联容器，红黑树实现，O(log n) 查找/插入/删除；
    • unordered_map 是哈希表，平均O(1)（最坏 O(n)）；
    • map 保证键值对按键升序，可做顺序遍历或查找邻近键；unordered_map 不排序但常用于频繁查找时效率更高。

以上各容器均提供类似的迭代器、begin()/end()、size()、clear() 等通用接口，但在性能和使用场景上有所区别。选择时请根据是否需要顺序、元素唯一性、操作类型（如随机访问 vs 插入删除频率）等因素决定。
